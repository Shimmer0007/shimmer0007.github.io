## 一、信息架构与板块层级分析

### 1.1 现有板块的本质分类

legacy 里有 12 个"App"，但它们绝对不是平等的。从**人因工程**角度，访客的注意力漏斗是：

```
① 我是谁？（建立信任/好奇心）
② 他在做什么？（价值展示）  
③ 深入了解某个方向  
④ 找到联系方式/探索彩蛋
```

按此映射，板块层级如下：

| 层级 | 板块 | 理由 |
|------|------|------|
| **L1 核心身份** | 关于我 | 整个站的锚点，访客第一需求 |
| **L2 智识输出** | 微光博客、课业笔记 | 最能体现主人思维深度的内容 |
| **L2 能力展示** | 技能树、项目浏览 | 投递/合作方的首要关注点 |
| **L3 生活切面** | 出行足迹、一周年纪念 | 立体人格展示，非功能性 |
| **L4 实用工具** | Hokie Ehall、GitHub查询 | 自用/特定场景，非核心展示 |
| **L4 系统功能** | 设置、友情链接 | OS 元功能 |
| **L5 时效性** | 二月计划 | 季节性内容，考虑改为"动态/Now" |

### 1.2 合并/拆分建议

**合并：**
- `微光博客` + `课业笔记` → **单一"写作"应用**，用 `category` 标签区分（Blog / Notes / Essays），减少认知负担
- `Hokie Ehall` + `GitHub查询` → **工具箱（Toolbox）**，作为一个抽屉式窗口收纳实用工具

**独立保留：**
- `技能树` — 可视化强，值得独立展示
- `项目浏览` — 有潜力做成精致的 Portfolio 展示
- `出行足迹` — 视觉独特，保留但降优先级

**升级改造：**
- `二月计划` → **「Now」** 页面（参考 Derek Sivers 的 /now 页），展示当前在做什么/读什么/想什么，赋予时效性，去掉月份硬编码

## 二、React 状态总线与联动协议

### 2.1 全局状态设计（Zustand）

```
GlobalStore {
  // OS 层
  theme: 'dark' | 'light'
  accentColor: string
  lang: 'zh' | 'en'
  
  // 窗口管理
  openWindows: WindowState[]
  activeWindowId: string | null
  
  // 内容联动（核心新增）
  activeTag: string | null        // 当前选中的标签
  activeSkillId: string | null    // 当前选中的技能节点
  readingHistory: string[]        // 最近阅读的文章 ID
  
  // "Now" 状态
  nowStatus: NowStatus            // 当前动态
}
```

### 2.2 跨窗口联动协议（虚拟事件总线）

这是最有趣的部分——打破 iframe 孤岛，让所有窗口真正互相认识：

```
联动场景举例：

📝 博客 → 🌱 技能树
  点击文章标签 "Python" 
  → 技能树高亮 Python 节点并展开相关技能
  → 侧边出现"相关文章"浮层

🌱 技能树 → 📂 项目
  点击技能节点 "数据可视化"
  → 项目列表自动筛选含该技能的项目
  → 两个窗口之间出现连接线动画（可选）

👤 关于我 → 📝 博客
  "关于我"下方的标签/兴趣点击
  → 博客窗口打开并筛选对应分类

📅 Now → 所有窗口
  "Now"状态中的 currentReading 字段
  → 博客窗口顶部出现"正在阅读"角标

🔍 全局搜索（新增）
  Cmd/Ctrl+K 呼出 Spotlight 式搜索
  → 搜索跨越所有窗口内容（文章、项目、技能）
  → 结果可直接定位到对应窗口
```

实现方式：所有 React 组件通过 Zustand store 订阅，不再用 `window.parent` 或 `postMessage`（彻底告别 iframe 地狱）。

## 三、配色方案：深宇宙 × 紫金撞色

### 3.1 色彩哲学

**紫** = 神秘感、技术深度、夜空  
**金** = 高光、重点、点睛  
两者的张力创造出"黑洞边缘"的视觉感——深邃但不沉闷。

### 3.2 设计令牌

### 3.3 呼吸感与动态效果

- **背景极光**：3个径向渐变光斑缓慢漂移，周期 20-30s，营造"活的宇宙"感
- **窗口 active 光晕**：active 窗口边框呼吸发光 `box-shadow: 0 0 30px var(--color-purple-glow)`
- **Dock 悬浮放大**：相邻图标随鼠标距离平滑缩放（macOS 风格，CSS + JS）
- **按钮微交互**：按下时 `scale(0.96)` + 轻微位移
- **标签/节点高亮**：跨窗口联动时，目标元素闪烁两次金色光晕

### 4.3 i18n 轻量方案

不使用重型 i18n 库，用自制 hook：

```typescript
// src/i18n/index.ts
export function useT() {
  const lang = useOSStore(s => s.lang);
  return (key: string, vars?: Record<string, string>) => {
    const dict = lang === 'en' ? en : zh;
    let str = dict[key] ?? key;
    if (vars) Object.entries(vars).forEach(([k, v]) => str = str.replace(`{${k}}`, v));
    return str;
  };
}
```

**原则**：优先中文内容，英文 key 若缺失 fallback 到中文，不阻塞开发。

## 五、主页桌面 — 人因工程布局

### 5.1 视觉重心设计

桌面不是等权的图标网格，而是有**权重的画布**：

```
┌─────────────────────────────────────────────────┐
│  ★ 核心 3 图标（L1-L2）— 居中偏左，略大尺寸      │
│  关于我 · 博客/写作 · 技能树                     │
│                                                  │
│  ○ 次级 3 图标（L2）— 核心右侧                   │
│  项目 · 出行足迹 · Now                           │
│                                                  │
│  · 工具类（L4）— 桌面右下角，略小，半透明         │
│  工具箱 · 友情链接 · 设置                        │
│                                                  │
│  ≡ Spotlight 入口 — 桌面顶部居中（类 macOS）     │
│                                                  │
│  ─────────────────── Dock ───────────────────── │
└─────────────────────────────────────────────────┘
```

**核心原则**：
1. 访客眼球路径：中央 → 右上 → 右下（F型 or Z型路径）
2. L1 图标用金色底纹微微发光做区分
3. 图标使用自定义 SVG，不用 Emoji
4. 悬浮图标时显示浮层预览（类似 macOS Quick Look）

### 5.2 开机动画升级

```
① 全屏深黑背景
② "shimmer" 文字逐字出现（字母渐入 + 向上位移）
③ 光晕从中心爆开（紫 → 金渐变）
④ 进度条出现（细线，紫金渐变，Spring动画）
⑤ 桌面图标以错开时序弹入（stagger animation）
⑥ Dock 从底部滑入
```
